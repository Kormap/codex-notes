# JPA Performance Review

## Workflow

1. Map entity relationships, fetch types, repository queries, and service access patterns.
2. Identify when lazy associations are accessed and how many SQL statements are produced.
3. Check paging, sorting, aggregation, bulk update/delete, and DTO projection paths.
4. Review transaction scope and persistence context size.
5. Recommend the lowest-risk fetch/query strategy.

## Checklist

- N+1 from lazy collections or `@ManyToOne` access in loops.
- `fetch join` with collection paging; prefer two-step query, batch size, or DTO projection.
- Overusing eager fetch; prefer query-specific fetch plan.
- `@EntityGraph` vs fetch join selection based on reuse and query complexity.
- Dirty checking on large loaded entity graphs.
- Bulk update/delete bypassing persistence context and stale entity state.
- OSIV hiding lazy loading from service boundaries.
- Missing indexes for foreign keys and query predicates generated by JPA.
